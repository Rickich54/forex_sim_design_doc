02 / Design philosophy

Layout decisions

The thinking behind every major design choice — from the sidebar layout to the use of cards and modular components.

A. Design philosophy
The goal of the interface

This platform needs to do three things at once: teach real trading skills, feel like a real trading tool, and keep the user engaged through game-like progression.

Every screen decision starts from this question: does this help the user learn faster without feeling like they are in a classroom?

The answer is a trading simulation interface where the environment itself is the teacher.

Why a trading platform style is used

Beginners who use a toy-looking interface never build real confidence. A realistic interface creates muscle memory for tools like MT5 and TradingView before the user ever touches real money.

The platform looks like Bloomberg or TradingView on the outside but behaves like a game on the inside. Difficulty scales, XP rewards good decisions, and mistakes become lessons rather than losses.

B. Why sidebar and dashboard layout

A sidebar keeps all modes visible at all times. The user never feels lost. Campaign, Arena, and Sandbox are always one click away.

The dashboard layout puts the most important information in the center and keeps secondary actions on the right. This matches how professional trading terminals are organized, so the skill transfers.

Why cards and modular components

Cards isolate one piece of information at a time. In a fast-decision environment, a crowded screen causes hesitation and errors.

Modular components also mean the same card design works on mobile and desktop. A market summary card looks the same whether it appears on the dashboard or inside the results screen.

Font choice

The platform uses Syne for all interface text — a geometric sans-serif that feels precise and modern without being cold. DM Mono is used for all code-like values: prices, XP numbers, timestamps, and labels.

Two weights only: 400 for reading, 500 or 600 for hierarchy. This keeps the interface calm and focused.

Why readability matters here

In a trading environment, a user may need to read a price, a risk level, and a news headline at the same time. Poor typography causes real mistakes.

Numbers use monospace figures so columns of prices align perfectly. P&L values always combine color with the number — never color alone — so colorblind users are not excluded.

Why dark mode is the default

Every professional trading terminal uses a dark interface. Bloomberg, MT5, and TradingView are all dark by default. Matching this aesthetic immediately signals to the user that this is a serious tool.

Dark backgrounds also make color signals like green and red stand out more clearly, which is important in a fast-decision environment.

Color meaning system

Blue means information: news cards, data points, and interface actions. Green means profit and correct analysis. Red means risk, loss, and warnings. Amber means caution and time pressure.

Purple is reserved for gamification only — XP, levels, and achievements. This keeps the game layer visually separate from the trading layer so users do not confuse the two.

05 / UX principles

UX principles

The five core principles that every design decision must support. When two design choices conflict, these principles decide which one wins.

E. Principles
Simplicity under complexity
The platform contains a lot of information: charts, orders, news, scores. The design hides complexity until the user needs it. A beginner sees a simple layout. An advanced user can expand panels and access deeper analytics. No feature is buried, but no feature shouts either.
Fast decision-making support
Every screen is designed around a single primary action. On the trading screen the primary action is placing a trade. On the event analysis screen it is selecting a direction. Secondary information supports the decision without competing for attention. The most important button is always the largest and most visible element on the screen.
Feedback-driven learning
Every action produces immediate, clear feedback. A good trade earns a green result card, an XP animation, and a short lesson about what made it correct. A bad trade earns a red result card, a clear explanation of the error, and a suggestion to improve. The user always knows why the outcome happened, not just what it was.
Progression visibility
The user can always see how far they have come and how far they have to go. The XP bar is visible in the sidebar on every screen. The campaign progress map shows completed and upcoming scenarios. Achievement badges are earned and displayed publicly. Progress never feels invisible.
Realism over simplification
When there is a choice between a simpler toy-like interface and a more realistic trading interface, the platform chooses realism. The goal is for users to feel ready to open a real account after completing the campaign. A simplified interface would defeat the entire purpose of the platform.

Post-trade feedback sequence
1
Outcome badge appears
Win, Loss, or Break-even badge appears large and centered. The user knows the result in under one second.
2
P&L counter animates
The profit or loss number counts up or down to the final value. Green for profit, red for loss.
3
Chart replay starts
The chart replays from entry to exit automatically, with clear markers showing where the trade opened and closed.
4
Lesson card slides in
A short lesson explains what happened and why. On a loss it gives a concrete improvement tip. On a win it reinforces the correct behavior.
5
XP animation fires
XP earned floats up from the result card and fills into the sidebar XP bar. The whole sequence takes about 4 seconds.

The level badge and XP bar live in the sidebar so they are always visible. When XP is earned, a floating counter animates from the trade result into the XP bar, which fills visibly. Level-up moments trigger a full-screen celebration, then return to the dashboard with the new level displayed. Every XP gain shows its source: correct analysis, good risk management, or a streak bonus.
